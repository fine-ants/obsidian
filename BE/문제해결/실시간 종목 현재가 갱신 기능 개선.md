
실시간 종목 현재가 갱신 기능 개선안

개요
- spring 서버에서 스케줄러를 이용하여 주식 장시간동안 5초간격으로 수행됨
- 한국투자증권 OPEN API에 질의하여 가격을 질의 및 응답받음
- 갱신되는 종목의 기준은 사용자들의 포트폴리오에 등록된 종목들입니다. 즉, 모든 종목(2700여개 정도)들을 갱신하는 것이 아닌 사용자의 포트폴리오에 등록된 종목들을 대상으로 한다.

문제점
- 사용자의 요청 처리와 종목 현재가 갱신 기능이 하나의 서버에서 처리하기 때문에 처리 성능이 낮아집니다.
- 사용자가 종목 가격을 요청하지도 않았는데 장 시간 동안에 계속 종목 가격 갱신을 처리하고 있습니다.

개발방향
- 종목 가격을 갱신하는별도의 모듈 서버로 분리하는 것을 고려하고 있음
- 사용자가 종목 가격과 관련된 API를 요청하면 최신 종목 가격을 제공하고 싶음
- 최신 종목 가격을 제공하면서도 사용자의 요청 처리는 빨라서 1초 미만(밀리초 단위)이어야 함
- 장시간 동안에 사용자가 요청하지도 않았는데 서버가 내부적으로 계속 종목 가격을 갱신하는 것은 원하지 않음

개선방안
1. 아키텍처 개선 : 서버 분리 및 역할 분담
	- Spring 서버에서 종목 가격 갱신 로직을 분리하여 별도의 서버에서 수행하도록 함
	- Stock Data Updater
		- 장중 종목 현재가 갱신 및 외부 API 통신
		- 비동기 스케줄링을 수행함
		- MQ(Message Queue)를 활용하여 갱신할 종목 리스트를 전달받음
2. 현재가 갱신 및 제공 로직 개선
	- 캐싱하기 위해서 Redis 사용
	- 종목 갱신 로직의 Lazy Update 도입
