
## 배경
- 전체 통합 및 단위 테스트 실행시 AOP 관련 테스트가 실패하는 것을 확인하였습니다.
- 에러 출력 확인한 결과 `@Secured` 애노테이션에 의한 AOP 실행시 SecurityContext의 인증 정보가 존재하지 않아서 인가 관련 에러가 발생한 것입니다.
- 그러나 ActiveStockAspectTest 통합 테스트 클래스만을 대상으로 테스트들을 실행시 테스트가 통과되는 상태입니다.

## 원인 분석
- 단일 통합 테스트 클래스 실행시 다음과 `@Secured` 애노테이션 기반한 AOP 실행시 다음과 클래스로 실행됩니다.
	- `@Secured` 애노테이션 기반 AOP 실행시 `AuthorizationManagerBeforeMethodInterceptor` -> `DeferringObservationAuthorizationManager` -> `ObservationAuthorizationManager` -> `SecuredAuthorizationManager` -> `AuthoritiesAuthorizationManager` 순으로 실행됩니다.
	- SecurityContextStrategy에 의해서 SecurityContext 조회시 최종적으로 `AuthorizationManagerBeforeMethodInterceptor`의 `getAuthentication()` 메서드에서 수행됩니다.
	- `this.securityContextHolderStrategy.get().getContext()` 호출시 반환값으로 인증정보가 포함된 객체가 반환된것을 확인할 수 있습니다.
![](refImg/Pasted%20image%2020260710125734.png)


- 모든 통합 테스트 클래스 실행시 `@Secured` 애노테이션 기반한 AOP 실행시 다음 결과와 같이 SecurityContext 조회하면 인증 정보를 저장하고 있지 않습니다.
	- `@Secured` 애노테이션 기반 AOP 실행시 `AuthorizationManagerBeforeMethodInterceptor` 클래스의 `getAuthentication()` 메서드를 통해서 인증정보를 가져옵니다.
	- `this.securityContextHolderStrategy.get()` 실행시 `ThreadLocalSecurityContextHolderStrategy`를 반환합니다.
	- 하지만 `this.securityContextHolderStrategy.get().getContext()` 조회시 반한된 SecurityContext 객체 안에는 인증정보가 저장되어 있지 않은 것을 볼수 있습니다.
